# Roadmap - e-Bujo

Planejamento de fases de desenvolvimento do projeto.

## Fases de Desenvolvimento

### Fase 1: Analog-First Foundation
**Status**: Em andamento

Foco na implementação da experiência básica do Bullet Journal, priorizando a simplicidade analógica.

- [ ] Setup inicial do projeto (React + TypeScript + Tailwind)
- [ ] Implementação do editor TipTap com símbolos BuJo básicos
- [ ] Sistema de autenticação
- [ ] Configuração do Supabase _(schema e RLS implementados no backend)_

- [ ] Estrutura básica de journals e entries _(schema pronto no backend; frontend a implementar)_

### Fase 2: Core Features
**Status**: Em planejamento

Implementação das funcionalidades essenciais do método BuJo.

- [ ] Future Log
- [ ] Monthly Log
- [ ] Daily Log
- [ ] Collections (custom collections)
- [ ] Migration (importação de dados analógicos)

### Fase 3: Digital-Native Enhancements
**Status**: Em planejamento

Adição de funcionalidades que aproveitam o potencial digital sem comprometer a simplicidade.

- [ ] Busca avançada
- [ ] Tags e filtros
- [ ] Templates personalizados
- [ ] Exportação (PDF, Markdown)
- [ ] Sincronização multi-dispositivo

### Fase 4: Refinamento e Otimização
**Status**: Em planejamento

Melhorias de performance, UX e funcionalidades avançadas.

- [ ] Otimização de performance
- [ ] Offline-first capabilities
- [ ] Analytics e insights pessoais
- [ ] Integrações externas (opcional)

---

## Princípios de Desenvolvimento

- **Analog-First**: Sempre priorizar a experiência analógica antes de adicionar recursos digitais
- **Simplicidade**: Cada feature deve passar pelo teste de "isso realmente melhora a experiência BuJo?"
- **Iteração Rápida**: Desenvolver em ciclos curtos, validando com usuários reais
